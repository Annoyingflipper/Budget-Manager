## Feature 1 â€” Projected-vs-Actual chart (page centerpiece)

For the selected month, render each category as paired bars: **projected** (budget) vs **actual** (spent).

- Over-budget (actual > projected) renders in the theme's "danger" red; on/under renders in the theme's "success" green.
- Data source: the `Budget` object from `getBudget(selectedMonth)` â€” already returns per-category `projected`/`actual` line items. The chart aggregates each category's items to category totals.
- Categories with zero projected and zero actual are omitted (nothing to show).
- Answers the core budgeting question â€” "am I on track?" â€” at a glance.

## Feature 2 â€” "vs last month" delta (automatic, zero controls)

Below the chart, a per-category comparison of the selected month's **actual** against the **previous month's actual**.

- "Previous month" = the calendar month before `selectedMonth` (e.g. viewing June 2026 â†’ compares to May 2026). Use the existing `src/utils/month.ts` helpers to compute it.
- Each row: category name + icon, this month's actual, and the delta with direction:
  - â–˛ red when this month spent **more**,
  - â–Ľ green when this month spent **less**,
  - "= same" (muted) when equal.
- Delta is on **actual** spend (the behavioral signal), category by category.
- **No-prior-data case:** if the previous month has no `line_items`/`income` rows, show a friendly "No prior month to compare" message instead of the delta table. (Use the previous month's *stored* data; do not fabricate a zero baseline that would make every category read as â–˛.)
- A category present in one month but not the other is treated as $0 on the missing side, and labeled accordingly.

## Feature 3 â€” CSV export

Two buttons on the Insights page:

- **Export this month** â€” line items for `selectedMonth`.
- **Export all history** â€” line items across every month from `listMonths()`.

Format: **one row per line item**, columns `Month, Category, Item, Projected, Actual`.

- Generated entirely client-side (build a CSV string, trigger a Blob download). No backend, no new dependency for export.
- **Escaping:** item/category names may contain commas, quotes, or newlines â€” wrap fields per RFC 4180 (quote fields containing `, " \n`, double any embedded `"`).
- Money values exported as plain numbers (e.g. `1650.00`), not formatted currency, so the file re-imports cleanly.
- Filenames: `budget-<month>.csv` (e.g. `budget-2026-06.csv`) and `budget-all-history.csv`.
