# Budget Manager v1.5.1 — Polish, changelog, analytics Spec

**Date:** 2026-05-30
**Status:** Approved — ready for implementation planning

---

## 1. Overview

v1.5.1 closes six items found during v1.5 QA smoke plus two follow-on adds: it makes the category drag-handle work on touch devices via a polyfill, refreshes the budget view in the background after any category mutation (with a small success toast), fixes the cramped "Confirm" button on `LineItemRow` and makes delete-confirm one-at-a-time, swaps the 🍑 logo to 💵, adds a Settings → "What's new" changelog modal that also auto-appears once per release, and turns on Vercel Analytics + Speed Insights.

It bundles on top of v1.5 on `staging`. The release ships as one merge to `main` (v1.5 + v1.5.1 together) so the user gets a single round of PRD smoke.

## 2. Goals

- **Mobile drag-and-drop:** install the `drag-drop-touch` polyfill (~2KB), import once in `src/main.tsx`. CategoriesEditor's existing HTML5 DnD wiring then works on touch with zero further changes.
- **Optimistic refresh + toast:** App owns a `refreshCounter` state and a single `toast` slot. CategoriesEditor receives an `onCategoriesChanged(action)` callback and fires it after every successful mutation. App bumps the counter (triggers a silent budget refetch via the existing useEffect) and shows a 2.5s success toast with copy keyed on `action`.
- **LineItem confirm UX (3a + 3b):** replace the inline "Confirm" text with a ✓ checkmark so it fits the existing 24px column. Lift `confirmingItemId: number | null` state from `LineItemRow` up to `CategoryTable` so only one row is ever in confirm state per category.
- **Changelog feature:** new `src/changelog.ts` module (hardcoded entries, newest first) and a `LATEST_VERSION` constant. New `<ChangelogModal />` component shown automatically on mount when `user_preferences.last_seen_changelog_version != LATEST_VERSION`, and also via a "🆕 What's new" button on the Settings page. Closing the modal upserts `last_seen_changelog_version = LATEST_VERSION`.
- **Logo:** swap 🍑 to 💵 in `Header.tsx`.
- **Analytics + Speed Insights:** mount `<Analytics />` and `<SpeedInsights />` inside the App tree. Both auto-detect production; no extra config.
- **Schema:** one column added to `user_preferences` (`last_seen_changelog_version text` nullable).
- **Tests:** ~7 new + light updates to existing `LineItemRow.test.tsx` and `CategoryTable.test.tsx`. Test count 70 → ~77.

## 3. Non-goals (v1.5.1)

- **Toast queue / multiple stacked toasts.** One slot; second toast overwrites first. Adding a queue is over-engineering for a solo app.
- **Undo toast actions.** Toasts are read-only confirmations.
- **Changelog rich content** (images, code blocks, embedded videos). Plain bullets per entry.
- **Per-version diff in the popup** (show every entry between last-seen and current). Show only the latest entry on auto-trigger; the "What's new" button shows the full list scrollable.
- **Vercel Analytics custom events / funnels.** Just install + mount; no `track(...)` calls.
- **Mobile DnD polish** (haptic feedback, drag-preview customization). Accept the polyfill defaults.
- **Keyboard-driven reorder for accessibility.** Still v1.6+.
- **Settings refactor** — keep the page as one file; we're just appending a "What's new" button at the bottom.

## 7. Components

### 7.1 `ChangelogModal`

Props: `{ entry: ChangelogEntry; onDismiss: () => void; showAllAfter?: ChangelogEntry[] }`.

- Full-screen dim background, centered card.
- Header: `What's new in v{entry.version}` + entry title + date.
- Body: bulleted list of `entry.highlights`. If `showAllAfter` is provided (manual "What's new" open), render each entry as a collapsible section below the headline entry.
- Footer: a single "Got it" button + X in the top-right.
- Backdrop click also dismisses.

### 7.2 `Toast`

Props: `{ message: string; type: 'success' \| 'error'; onDismiss: () => void; timeoutMs?: number }`.

- Fixed `bottom: 16px; left: 50%; transform: translateX(-50%)`.
- `bg-positive/90` for success, `bg-negative/90` for error.
- `useEffect` registers a `setTimeout` that calls `onDismiss` after `timeoutMs ?? 2500`.

### 7.3 LineItemRow controlled confirm

```tsx
type Props = {
  item: LineItem;
  isConfirming: boolean;
  onConfirmRequest: () => void;
  onChange: (next: LineItem) => void;
  onDelete: () => void;
};
```

- `isConfirming === false` → render `<button aria-label="Delete row" onClick={onConfirmRequest}>✕</button>`.
- `isConfirming === true` → render `<button aria-label="Confirm delete" onClick={handleDelete}>✓</button>`.
- `handleDelete` calls `onDelete()` then `deleteLineItem(item.id)` — the existing API flow.

### 7.4 CategoryTable confirm coordinator

`const [confirmingItemId, setConfirmingItemId] = useState<number | null>(null);`

Each row gets `isConfirming={confirmingItemId === item.id}` + `onConfirmRequest={() => setConfirmingItemId(item.id)}`. On successful `removeItem`, clear `confirmingItemId`.

### 7.5 `CategoriesEditor` change-callback wiring

New prop:

```ts
type Props = {
  onCategoriesChanged?: (action: 'added' | 'renamed' | 'icon' | 'deleted' | 'reordered') => void;
};
```

Calls it in five places: after `addCategory` resolves, after `reorderCategories` resolves, after `confirmDelete` resolves, and (through a new optional `CategoryRow.onCategoryChanged` callback) after `renameCategory` or `setCategoryIcon` resolves.

### 7.6 App-level wiring

```tsx
const [refreshCounter, setRefreshCounter] = useState(0);
const [toast, setToast] = useState<{ message: string; type: 'success' | 'error' } | null>(null);
const [changelogOpen, setChangelogOpen] = useState(false);
const [lastSeen, setLastSeen] = useState<string | null>(null);

useEffect(() => {
  // ... existing getBudget + listMonths
}, [selectedMonth, bootstrapped, refreshCounter]);

useEffect(() => {
  // On mount: read last_seen_changelog_version.
  getLastSeenChangelogVersion().then((v) => {
    setLastSeen(v);
    if (v !== LATEST_VERSION) setChangelogOpen(true);
  }).catch(() => {});
}, []);

const TOAST_COPY: Record<string, string> = {
  added:     'Category added.',
  renamed:   'Category renamed.',
  icon:      'Icon updated.',
  deleted:   'Category deleted.',
  reordered: 'Order saved.',
};

const handleCategoriesChanged = useCallback(
  (action: 'added' | 'renamed' | 'icon' | 'deleted' | 'reordered') => {
    setRefreshCounter((n) => n + 1);
    setToast({ message: TOAST_COPY[action], type: 'success' });
  },
  [],
);

async function dismissChangelog() {
  setChangelogOpen(false);
  try { await setLastSeenChangelogVersion(LATEST_VERSION); }
  catch { /* non-fatal; will re-prompt next session */ }
}
```

## 8. `src/api/userPrefs.ts` (new)

```ts
import { supabase } from '../lib/supabase';

async function currentUserId(): Promise<string> {
  const { data } = await supabase.auth.getUser();
  if (!data.user) throw new Error('Not authenticated');
  return data.user.id;
}

export async function getLastSeenChangelogVersion(): Promise<string | null> {
  const userId = await currentUserId();
  const { data, error } = await supabase
    .from('user_preferences')
    .select('last_seen_changelog_version')
    .eq('user_id', userId)
    .maybeSingle();
  if (error) throw error;
  return data?.last_seen_changelog_version ?? null;
}

export async function setLastSeenChangelogVersion(version: string): Promise<void> {
  const userId = await currentUserId();
  const { error } = await supabase
    .from('user_preferences')
    .update({ last_seen_changelog_version: version, updated_at: new Date().toISOString() })
    .eq('user_id', userId);
  if (error) throw error;
}
```
