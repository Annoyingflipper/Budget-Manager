# Budget Manager v1.5 — Custom user-editable categories Spec

**Date:** 2026-05-30
**Status:** Approved — ready for implementation planning

---

## 1. Overview

v1.5 lifts the eight categories out of the seed trigger and makes them yours to edit. You can **rename** ("Loans" → "Mortgage"), **add** new ones, **delete** with a forced move of any existing items to a category you pick, **reorder** with a drag handle, and **pick the emoji** for each one. Categories stay user-level (one set across all months) so an added or renamed category shows up in past and future months alike.

The editor lives in a new **Settings → Categories** section. The budget view is otherwise unchanged: same `CategoryTable`, same icon-in-the-header rendering — just reading `category.icon` directly off the row instead of a name-keyed lookup table. The keyword-matching utility `src/utils/categoryIcons.ts` gets deleted.

Two new Postgres RPCs do the atomic work that a single SQL statement can't: `move_and_delete_category(src, dst)` moves all items then deletes the category in one transaction, and `reorder_categories(ordered_ids[])` updates `display_order` for the full set in one shot. The FK from `line_items.category_id` to `categories.id` flips from `ON DELETE CASCADE` to `ON DELETE RESTRICT` so a direct delete that skips the RPC fails loudly instead of silently destroying history.

## 2. Goals

- Add a `categories.icon text NOT NULL DEFAULT '📁'` column; backfill the eight existing categories with their original emojis from `src/utils/categoryIcons.ts`.
- Change `line_items.category_id` FK from `CASCADE` to `RESTRICT` so direct deletes can't orphan-delete line items.
- Add `move_and_delete_category(src_id, dst_id)` Postgres RPC with AAL2 guard, atomic move + delete in one transaction, and a check that prevents deleting the user's last category.
- Add `reorder_categories(ordered_ids bigint[])` Postgres RPC with AAL2 guard and a check that the array covers every category the user owns.
- Update the seed trigger so new signups get categories with explicit emoji values.
- New API module `src/api/categories.ts` exporting `addCategory`, `renameCategory`, `setCategoryIcon`, `moveAndDeleteCategory`, `reorderCategories`.
- New Settings → Categories section with `CategoriesEditor`, `CategoryRow`, `EmojiPicker` components.
- Delete dialog that forces a destination pick when the source category has any items (across any month).
- Native HTML5 drag-and-drop reorder. No new runtime dependency.
- Delete `src/utils/categoryIcons.ts` and replace its only consumer (`CategoryTable.tsx`) with a direct `category.icon` read.
- ~13 new tests; total 56 → ~69.

## 3. Non-goals (v1.5)

- **Bulk operations.** No multi-select delete, no "move all items from X to Y" outside the delete-dialog path. Single-row CRUD is sufficient.
- **Category-level budgets** ("Entertainment capped at $200/month"). Separate feature (Goals); revisit in v1.7+.
- **Sharing categories with another user.** Sharing arrives in v1.6.
- **Per-month-overridden categories.** Explicit anti-goal from v1.4 — categories remain user-level.
- **Custom-image upload for icons.** Curated emoji grid + paste-any-emoji is enough.
- **Soft delete / restore / "trash bin".** Full delete is enough because the RPC preserves the items.
- **Keyboard-driven reorder** (arrow keys on the drag handle). Pointer-only is enough for an 8–15-row list.
- **Undo for delete.** Confirm dialog is the safety net; recreate by hand if you regret.
- **Editing the eight category-name uniqueness rule.** `(user_id, name)` unique constraint from `0001` stays; duplicate names raise at the DB level and the UI surfaces the error.

### 5.2 `0013_categories_rpc.sql`

```sql
create or replace function public.move_and_delete_category(src_id bigint, dst_id bigint)
returns void
language plpgsql
security invoker
set search_path = public
as $$
declare
  uid uuid := auth.uid();
begin
  if uid is null then raise exception 'not authenticated'; end if;
  if (select auth.jwt() ->> 'aal') <> 'aal2' then raise exception 'mfa required'; end if;
  if not exists (select 1 from public.categories where id = src_id and user_id = uid)
     or not exists (select 1 from public.categories where id = dst_id and user_id = uid) then
    raise exception 'category not found';
  end if;
  if src_id = dst_id then raise exception 'src and dst must differ'; end if;
  if (select count(*) from public.categories where user_id = uid) < 2 then
    raise exception 'cannot delete the last category';
  end if;

  update public.line_items set category_id = dst_id
    where user_id = uid and category_id = src_id;
  delete from public.categories where id = src_id and user_id = uid;
end;
$$;

create or replace function public.reorder_categories(ordered_ids bigint[])
returns void
language plpgsql
security invoker
set search_path = public
as $$
declare
  uid uuid := auth.uid();
begin
  if uid is null then raise exception 'not authenticated'; end if;
  if (select auth.jwt() ->> 'aal') <> 'aal2' then raise exception 'mfa required'; end if;
  if (select count(*) from public.categories where user_id = uid) <> array_length(ordered_ids, 1) then
    raise exception 'must reorder every category';
  end if;

  update public.categories c
     set display_order = i.idx
    from unnest(ordered_ids) with ordinality as i(id, idx)
   where c.id = i.id and c.user_id = uid;
end;
$$;

grant execute on function public.move_and_delete_category(bigint, bigint) to authenticated;
grant execute on function public.reorder_categories(bigint[]) to authenticated;
```

## 7. UI

### 7.1 Settings → Categories section

`Settings.tsx` already has an Appearance section. Add a sibling section below it titled "Categories" with helper text *"Drag to reorder. Click an emoji to change it."* and a `<CategoriesEditor />` mounted underneath.

### 7.2 `CategoriesEditor.tsx`

- Fetches `categories` on mount via a direct supabase query (or via a `useCategories` hook — implementation detail).
- Renders a vertical list of `<CategoryRow />`. Owns the local categories array.
- HTML5 drag-and-drop wiring: `draggable`, `onDragStart` (sets dragged id), `onDragOver` (preventDefault), `onDrop` (computes new index, reorders local array, calls `reorderCategories([…ids])` optimistically — reverts on error).
- "+ Add category" button at the bottom. Click → appends a draft row (empty name, default `📁` icon, autofocus). On blur with a non-empty name → calls `addCategory`, replaces draft with the returned `Category`. Empty name → discards.
- Owns the delete dialog state. When a `CategoryRow`'s delete button fires, shows the modal scoped to that source category.

### 7.3 `CategoryRow.tsx`

Props: `{ category, onChange, onDelete, drag handlers }`.

Layout (desktop):

```
⋮⋮  <emoji>  <name input ──────>  🗑
```

- Drag handle `⋮⋮` on the left — only this element gets `draggable={true}` so dragging from inside the name input doesn't trigger a row drag.
- Emoji button — opens `EmojiPicker` popover when clicked. On pick, calls `setCategoryIcon(id, emoji)` optimistically.
- Name input — `save-on-blur` pattern matching `IncomeSummary`/`LineItemRow`. Empty name → revert. Same name → no-op. Otherwise call `renameCategory(id, name)`.
- Delete button (🗑) — fires the parent's `onDelete(category)`.

Mobile: stack vertically inside the row (drag handle + emoji on one row, name on the next, delete in the corner). Reuse `useIsMobile`.

### 7.4 `EmojiPicker.tsx`

- Triggered by clicking a row's emoji button. Popover positioned below the button.
- 6×5 grid of 30 curated emojis: `🛠 🎬 🏦 📋 💎 🧾 🧴 ✨ 🍔 🚗 🏠 🏥 ✈️ 🛒 📱 🎮 📚 👕 💊 🐕 🎁 ⚽ ☕ 🎨 💼 🏋️ 🎓 🎵 🌱 ⛽`.
- Bottom row: freeform text input — "Or paste any emoji". Submitting (Enter) picks the first emoji-length character of the input. Empty input is ignored.
- Clicking outside the popover, or pressing Escape, closes without picking.

### 7.5 Delete dialog

When 🗑 is clicked on a category, parent counts items across all months for that category (one SELECT) and decides which dialog to show:

- **Has items** (1+ across any month):

  > **Delete "Loans"?**
  > This category has 17 items across 3 months.
  > Move them to: \[ Other ▾ \]
  > \[ Cancel \] \[ Move & delete \]

  Dropdown contains every other category. Confirm → `moveAndDeleteCategory(srcId, dstId)`. On success, remove from local list and refetch budget on next budget-page load. On error, show the message.

- **No items**:

  > **Delete "Loans"? It has no items.**
  > \[ Cancel \] \[ Delete \]

  Confirm → direct `supabase.from('categories').delete().eq('id', X)`. (The new RESTRICT FK protects us if a stray item shows up between the count and the delete — the call would raise.)

### 7.6 Budget view

`CategoryTable.tsx`: change one line — `<span>{categoryIcon(category.name)}</span>` becomes `<span>{category.icon}</span>`. Drop the `categoryIcon` import.

`EmptyCategoryCard.tsx`: add a `categoryIcon: string` prop and render it directly; drop the `categoryIcon` import. Update the single callsite in `CategoryTable.tsx` to pass `categoryIcon={category.icon}`.
